<h1>Radio</h1>

<h3>What is</h3>
<p>
  The Radio component is a UI element used to present a set of options where
  users can select only one choice from multiple options.
</p>

<h3>How it works</h3>
<p>
  The Radio component consists of individual radio buttons, each representing a
  single option. Users can select one option by clicking on the respective radio
  button.
</p>
<Primary />

<h3>Best Practices</h3>
<ul>
  <li>
    Group related radio buttons using the same `name` attribute to create a
    mutually exclusive selection.
  </li>
  <li>
    Provide clear and concise labels for each radio button to assist users in
    understanding their options.
  </li>
  <li>
    Ensure proper alignment and spacing between radio buttons for better visual
    clarity.
  </li>
</ul>

<h3>Props</h3>
<Controls />

<h3>Accessibility</h3>
<ul>
  <li>Use semantic HTML elements (`<fieldset>`, `<legend>`) to group related radio buttons for improved accessibility.</li>
  <li>Associate each radio button with its corresponding label using the `for` attribute on the label and matching `id` on the input element.</li>
  <li>Ensure the radio buttons are operable via keyboard navigation and provide clear focus states.</li>
</ul>

<h3>Do</h3>
<ul>
  <li>
    Use radio buttons when users need to select only one option from a set of
    choices.
  </li>
  <li>Provide descriptive labels for each radio button.</li>
  <li>Group related radio buttons using the same `name` attribute.</li>
</ul>

<h3>Don'ts</h3>
<ul>
  <li>
    Avoid using radio buttons when users need to select multiple options
    simultaneously.
  </li>
  <li>Don't forget to provide labels for each radio button for clarity.</li>
</ul>
